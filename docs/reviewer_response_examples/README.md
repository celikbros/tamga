# Reviewer Response Examples

This folder contains fictional examples. They show what a useful professor or
PhD-student response might look like after receiving the multilingual/Turkic
review packet.

These are not real reviews and should not be treated as project evidence.

## Do Reviewers Need Both Turkish and English?

Ideal: yes, especially for multilingual/Turkic reviewers.

Reason:

- Turkish helps them understand the Turkish-centered goal.
- English helps them read technical terms such as `byte fallback`, `script-aware
  pretokenizer`, `Unicode normalization`, and `vocabulary allocation`.
- Turkic-language reviewers may also need expertise in Azerbaijani, Kazakh,
  Kyrgyz, Uzbek, Tatar, or another relevant language/script.

But this is not a hard requirement for every role.

| Role | Turkish | English | Other expertise |
| --- | --- | --- | --- |
| Turkish hidden eval labeler | Required | Helpful, not mandatory | Turkish morphology |
| Turkish hidden eval reviewer | Required | Helpful | Linguistics/NLP |
| Multilingual/Turkic architecture reviewer | Helpful | Strongly helpful | Language/script/tokenization expertise |
| Turkic-language reviewer | Helpful | Helpful | Target Turkic language/script |
| European-language reviewer | Not required | Required or local language + technical NLP | Apostrophe/script/normalization knowledge |

If a strong reviewer does not read English well, prepare a Turkish summary of
the relevant docs before asking for feedback.
